Gladly WebGL Crash Tests

Open each test one at a time. Report which ones crash the entire browser (not just the tab).

  • 01 — desynchronized: true context
    Creates a WebGL2 context with { desynchronized: true } and runs a simple render loop.
  • 02 — Long-running single draw call
    Fullscreen quad with a 500K-iteration fragment shader. Tests Windows TDR threshold (~2s GPU timeout).
  • 03 — Many sequential FBO passes
    64 sequential framebuffer ping-pong passes with texture reads. Simulates FFT compute pipeline.
  • 04 — 1M-point additive blend scatter
    Draws 1 million points with additive blending. Simulates histogram scatter pass.
  • 05 — RGBA+FLOAT texImage2D format
    Calls texImage2D with RGBA (not RGBA32F) as internal format + FLOAT type. Tests invalid WebGL2 format handling.
  • 06 — Gladly ComputePipeline data generation
    Runs the full 1 M-point GPU data generation (13 columns) and CPU readback in isolation — no plot rendering. Times each column readback individually to pinpoint which step causes crashes.
  • 07 — Gladly 5 concurrent plot initializations
    Creates 5 plots simultaneously (3 scatter + 2 histograms with 1 M additive-blend scatter each). If 06 passes but 07 crashes, the bug is concurrent GPU command pile-up across multiple plots/contexts.
