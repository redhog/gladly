Test 16: Merged vs separate 2D colorscale functions in one shader

Compiles two variants of the full ColorbarLayer2d-equivalent fragment shader using raw WebGL2 (bypassing regl/gladly) to measure timing and success/failure directly.

  • Variant A — current (separate): 9 individually-defined colorscale_2d_NAME(vec2 t) functions + map_color_2d that calls them by index. Same as the failing test 12.
  • Variant B — merged: No separate function definitions. All 9 colorscale bodies inlined directly into a single map_color_2d(int cs, vec2 t) function as if/else branches. Identical total GLSL code, but only one function definition instead of 10.

If A fails and B passes: it is the COUNT of function definitions, not the amount of code. Merging is a viable fix.

If both fail or both pass: it is the total code amount; merging does not help.
