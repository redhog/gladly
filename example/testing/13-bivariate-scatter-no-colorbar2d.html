Test 13: Scatter with vData + vData2 but NO 2D colorbar

Scatter layer with both vData and vData2, using 1D colorscales only for both axes. No colorbars entry. colorbars: [].

Theory tested: Because both vData and vData2 are present, colorAxis2dQuantityKinds = {'': ['','2']} is set on the layer (ScatterShared.js:30), making map_color_s_2d → map_color_2d → all 9 2D colorscale functions reachable in the scatter shader's call graph — even though both cs_a and cs_b are positive 1D indices at runtime and the 2D branch is never taken.

Compare with test 08 (which also has a 2D colorbar). If this crashes like 08, the scatter shader alone is sufficient. If it passes, the ColorbarLayer2d is needed too.
